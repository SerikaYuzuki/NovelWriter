# Phase 4 実行計画: 小説執筆支援機能

Phase 4 の詳細なサブフェーズ分解。**実行エージェント(Codex 等)はこのファイルを作業指示書として使うこと。**
前提知識は [../AGENTS.md](../AGENTS.md)(必読)、設計は [DESIGN.md](DESIGN.md)、決定記録は [DECISIONS.md](DECISIONS.md)。

## 進め方の共通ルール

- **1サブフェーズ = 1ブランチ = 1PR**。着手順は 4-1 → 4-2 → 4-4 を必須の背骨とし、4-3 / 4-5 / 4-6 はその後に任意の順で
- 各サブフェーズ内のコミットは「モデル(+テスト)→ 保存層(+テスト)→ UI」の順の意味単位で
- PR 前に `./Scripts/check.sh` が「All checks passed」まで通ること(lint violation ゼロ含む)
- モデル・保存層の変更には必ず swift-testing のテストを書く(round-trip、後方互換、エッジケース)
- UI 文言は日本語。再利用可能な部品は NovelUI へ
- サブフェーズ完了時に、この文書のチェックボックスと DESIGN.md 11章の状態を更新し、新しい設計判断があれば DECISIONS.md に D-XXX として追記する
- 保存形式の詳細を App 層に漏らさない(DESIGN 9.3)。パッケージ内部への操作が必要な機能(添付など)は、必ず NovelCore の抽象プロトコル + NovelStorage の実装として設計する

## データ配置の全体像(D-018 で決定済み)

```text
MyNovel.novelpkg/
├── manifest.json          formatVersion "2"(4-1 で "1" → "2")
├── chapters/<ChapterID>.md   本文(既存)
├── notes/<ChapterID>.md      章メモ(4-1。空メモはファイルを作らない)
├── characters.json           キャラクター(4-2。配列順 = 表示順)
├── plot.json                 シーンカード(4-4。配列順 = 表示順)
├── flags.json                伏線・フラグ(4-5)
├── attachments/              資料(既存。4-6 でUI)
└── snapshots/                スナップショット(既存)
```

- 読み込み: formatVersion "1" と "2" を受理。欠けているファイルは「空」として読む(データ救出優先の既存方針と同じ)
- 保存: 常に "2" で書く。**パッケージ直下の未知のファイル/ディレクトリは保存時に保持する**(4-1 で実装。将来のフォーマット追加に対する安全弁)
- 各 JSON は prettyPrinted + sortedKeys(git 差分に優しく)
