## この計画で解決すること

- 作品を安全に新規作成・開く・別名保存でき、保存の成功／失敗が画面上で分かる
- 書き出しを `.novelpkg` の内部形式から独立させ、現在の原稿の不変スナップショットを安全に出力する
- v1 の出力としてプレーンテキスト、Markdown、EPUB 3、PDF を提供する

「高度な組版」「縦書き」「ルビ」「画像を含む EPUB」「複数作品同時編集」「クラウド同期」はこの計画の非目標とする。AI 支援にも着手しない。

## 全体方針

### 先に安定化を入れる理由

現状は自動保存・終了前保存・スナップショット・添付の直列化ができており、基盤は良好である。一方で、通常操作としての新規作成／既存作品を開く／別名保存の入口がなく、保存失敗はログ出力のみで UI が常に「自動保存」と表示する。出力を追加して配布可能な道具にする前に、ここを解消する。

また、通常保存はパッケージ全体を一時ディレクトリへ書き出し、添付と snapshots を引き継ぐ。大きい資料や多量のスナップショットで自動保存が遅くなる可能性があるため、推測で保存方式を作り替えず、まず測定し、予算を定めてから対処する。

### 出力の境界

Phase 5 では `NovelKit` に `NovelExport` ターゲットを追加する。依存方向は次のとおりとし、出力機能が保存形式や App の状態を知ることを禁止する。

```text
NovelApp (コマンド、保存先パネル、進捗・結果表示)
    ↓
NovelExport (出力仕様、テキスト／EPUB／PDF の生成)
    ↓
NovelCore (NovelDocument)

NovelStorage (.novelpkg の読み書き) ──→ NovelCore
```

- `NovelExport` は `NovelCore` のみに依存し、`.novelpkg` のパスや manifest を読まない
- 公開 API は `NSTextView`、`NSPrintOperation` などの AppKit 型を出さない。PDF の macOS 実装は `Platform/macOS/` と `#if canImport(AppKit)` に閉じ込める
- 出力は呼び出し時の `NovelDocument` 値を入力にする。出力中の編集はブロックせず、その後の編集を出力へ混ぜない
- 生成物は一時 URL に書き、成功時だけ目的地を置き換える。失敗・キャンセルで既存の出力を壊さない
- 出力は本文・作品名・章名・話名だけを対象にする。メモ、人物、プロット、伏線、添付、スナップショットは v1 の出力に含めない

## 先に固定する出力仕様

| 形式 | v1 の仕様 | 非目標 |
| --- | --- | --- |
| プレーンテキスト | UTF-8 (BOM なし)、LF 改行。`【作品名】`、`■ 章名`、`● 話名`の3階層を出し、話本文を順に連結する | 独自の原稿用紙記法、縦書き指定 |
| Markdown | UTF-8、`# 作品名`、`## 章名`、`### 話名`。本文はユーザーが書いた Markdown 互換テキストをエスケープせずに保持する | 独自 front matter、メモ／カードの埋め込み |
| EPUB | EPUB 3 のリフロー型、`ja` メタデータ、目次、章ごとの XHTML。章を `h1`、話を `h2` とし、本文は HTML エスケープして段落化する | 画像・添付の埋め込み、縦書き、ルビ、高度な CSS テーマ |
| PDF | 横書き A4、固定余白、本文用明朝体、章見出し、話見出し、ページ番号。日本語・絵文字を含む本文を壊さない | エディタ画面の完全再現、印刷所向け組版、縦書き |

全形式に共通する原稿展開規則は次のとおりとし、形式別レンダラが独自に章／話を走査しない。

- 順序は `NovelDocument.chapters`、続いて各 `Chapter.episodes` の配列順。`Chapter` から本文を取得する互換 API は設けない
- 見出し名は前後の空白と改行を除いて空なら、作品=`無題の作品`、章=`無題の章`、話=`本文`を表示名として使う。モデル値自体は変更しない
- 本文内の CRLF / CR は LF へ正規化する。本文先頭の空白・全角スペース、本文中の空行、末尾以外の改行は保持する
- 空章は章見出しだけを出す。空話は話見出しだけを出す。存在しない話や本文を捏造しない
- テキスト / Markdown は、見出しと本文ブロックの間を空行1つで区切り、ファイル末尾をLFちょうど1つに正規化する
- EPUB / PDF は章単位で改ページする。章内の話は配列順に続け、空話でも見出しを表示する

EPUB と PDF の出力スタイルは `EditorSettings` に依存させない。将来の書き出し設定として独立させ、初期版は上表の固定プリセットだけを提供する。
