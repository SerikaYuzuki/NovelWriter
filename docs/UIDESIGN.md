# UI刷新 実行計画(Phase UI)

GUI を「機能が全部ある」状態から「執筆・キャラ設計・プロット構成のそれぞれが気持ちよくできる」状態へ刷新する。
**実行エージェント(Codex 等)はこのファイルを作業指示書として使うこと。** 前提は [../AGENTS.md](../AGENTS.md)、設計は [DESIGN.md](DESIGN.md)、決定は [DECISIONS.md](DECISIONS.md)(特に D-019)。

## 設計方針(決定済み・変更しない)

### 単一ウィンドウ・3モード制

現在の「執筆画面の右インスペクタに5タブを詰め込む」構造をやめる。260pt の縦長パネルはキャラクター設計にもプロット構成にも狭すぎ、作業の「モード」が違うものを同居させているのが GUI が微妙な根本原因。

```text
ツールバー: [執筆 | キャラクター | プロット]  ← segmented control, Cmd+1/2/3
─────────────────────────────────────────────
執筆モード:        章リスト | エディタ+ステータスバー | インスペクタ(章コンテキスト)
キャラクターモード: キャラ一覧 | キャラクターシート
プロットモード:     カードボード(章レーン) | 伏線トラッカー
```

- モードはウィンドウ1枚の中身の切り替え(`AppMode` enum を AppState に持ち、UserDefaults に永続化)
- 参考にした定番UX: Nola(テンプレート項目式キャラシート・作家向け国内定番)、Scrivener(corkboard: シーン=カード)、Plottr(レーン式の視覚的プロット)

### プロットUI = 章レーン式カードボード

Scrivener の corkboard(カード操作)と Plottr のレーン概念の折衷。**レーン(縦列)= 章、カード = シーン/展開**。先頭に「未割り当て」レーンを置く。既存モデル `PlotCard { title, memo, chapterID? }` と1:1で対応し、**モデル変更なしで成立する**のが採用理由。カードのレーン間ドラッグ = `chapterID` の付け替え、レーン内ドラッグ = 順序変更。

### キャラクターUI = シート型(テンプレート項目 + 自由記述)

Nola 系の定番。左に一覧、中央に構造化されたプロフィールシート。項目はテンプレート固定(下記)+ 自由メモ。カスタム項目の追加は将来(スコープ外)。関係図も将来。

## UI-1: 基盤リファクタ + モード骨格 【必須・最初】

見た目の変化は小さいが、以降の全サブフェーズの土台。

### Must

- [ ] **ツールバー整理**: 現状の詰め込み(検索欄+ボタン6個)を解消する
  - 検索フィールドを常駐から撤去し、**Cmd+F でエディタ上部にスライド表示される検索バー**へ(閉じる: Esc。前へ/次へ/見つかりませんの表示は現行機能を維持)

  - スナップショット保存をメニューバー「ファイル > スナップショットを保存」(Cmd+Shift+S)へ移動し、ツールバーから撤去

- [ ] メニューバー整備: 上記スナップショットのほか、「表示」メニューにモード切替(Cmd+1/2/3)を登録

### Nice to have

## UI-2: キャラクターモード(シート型) 【必須】

### レイアウト

```text
┌─ キャラ一覧(220pt) ─┬─ キャラクターシート(残り全部) ─────────┐
│ ● 山田太郎           │  [カラー] 名前 [ふりがな]                │
│ ● 月宮ヒロイン       │  ── 基本 ──────────────                │
│   (+ 追加 / 並べ替え) │  役割 [主人公▾] 年齢 [17] 性別 [女]      │
│                      │  ── 口調 ──────────────                │
│                      │  一人称 [私] 二人称 [あなた]             │
│                      │  口調・話し方 [複数行]                   │
│                      │  ── 設定 ──────────────                │
│                      │  外見 / 性格 / 背景・経歴 [各複数行]      │
│                      │  ── 自由メモ ── [複数行]                │
│                      │  ── 登場章(自動検出) ── [章名リスト]    │
└──────────────────────┴──────────────────────────────────────┘
```

### Must

- [ ] **モデル拡張**(NovelCore `Character`): 全て Optional で追加 — `colorHex: String?`, `role: String?`, `age: String?`(数値でなく自由記述), `gender: String?`, `firstPerson: String?`, `secondPerson: String?`, `speechStyle: String?`, `appearance: String?`, `personality: String?`, `background: String?`。`decodeIfPresent` で後方互換、nil はエンコード時にキー省略。既存 characters.json が読めるテスト+round-trip テスト
- [ ] **一覧**(左): カラードット+名前+ふりがな。追加・削除(確認)・並べ替え(既存ヘルパー使用)
- [ ] **シート**(中央): 上記レイアウトの Form。役割はテキスト入力+定番候補のメニュー(主人公/ヒロイン/ライバル/敵役/脇役/モブ)。カラーは ColorPicker(colorHex と相互変換のユーティリティは NovelUI に置きテスト)
- [ ] **登場章**: 既存の 4-3 検出ロジックを移設・再利用。クリックで執筆モードに切り替えて該当章・該当位置へジャンプ
- [ ] 保存は既存経路(markDirty + デバウンス)。シートの各フィールドは編集で即モデル反映
- [ ] 執筆モードのインスペクタから「キャラクター」タブを撤去(本体はこのモードへ移設)

### Nice to have

- [ ] 一覧のカラードットを章リスト・プロットカードにも波及(登場キャラの視覚的紐付け)
- [ ] 名前の重複警告(PHASE4 積み残し)

### 完了条件

キャラの作成→シート記入→本文へのジャンプが1画面で回る。旧データが欠損なく読める。check.sh 全通過。

## UI-3: プロットモード(章レーンボード + 伏線トラッカー) 【必須】

### レイアウト

```text
┌─ カードボード(横スクロール) ──────────────────────┬─ 伏線(260pt) ─┐
│ [未割り当て] [第1章]      [第2章]      [第3章] …   │ ▼ 未回収 (2)   │
│ ┌────────┐  ┌────────┐   ┌────────┐              │  ・銀の懐中時計 │
│ │カードA  │  │カードB  │   │カードD  │              │  ・母の手紙     │
│ └────────┘  │カードC  │   └────────┘              │ ▶ 回収済み (5) │
│  [+]        └────────┘    [+]                     │  [+ 伏線を追加] │
└───────────────────────────────────────────────────┴────────────────┘
```

### Must

- [ ] **ボード**: 横スクロールの LazyHStack。レーン = 「未割り当て」+ 章(章順)。レーンヘッダ = 章タイトル+カード数+[+](その章に紐付くカードを追加)。カード = タイトル+メモ冒頭(2行まで)
- [ ] **ドラッグ&ドロップ**: `PlotCardID` を `Transferable` にし、`.draggable` / `.dropDestination` で (1) レーン間移動 = `chapterID` 付け替え (2) レーン内並べ替え。**挿入位置の計算はモデルヘルパーに切り出す**: `NovelDocument.movePlotCard(id:toChapter:before:)` のような API を NovelCore に追加し、「配列全体は1本の順序、レーン内の順序は配列順の射影」という不変条件を保つ。境界ケース(空レーンへ、同レーン内で自分の直後へ、未割り当てへ)をテストで固定
- [ ] **カード編集**: カード選択で詳細ポップオーバー(またはシート): タイトル / メモ / 章ピッカー / 削除(確認)。Enter=編集開始、⌫=削除(確認)くらいのキーボード対応
- [ ] **伏線トラッカー**(右パネル): 現行インスペクタの伏線タブを移設。未回収を上・回収済みは折りたたみ、回収チェック、張った章/回収章へのジャンプ(執筆モードへ切替+ジャンプ)
- [ ] 執筆モードのインスペクタから「プロット」「伏線」タブを撤去(移設)

### Nice to have

- [ ] カードに紐付く登場キャラのカラードット表示(UI-2 の Nice と連動)
- [ ] カードのステータス(未着手/執筆中/済み)と色分け(PHASE4 積み残し。やる場合はモデルに `status` を Optional 追加+テスト)

### 完了条件

ボード上でカードを作り、章に配って並べ替える操作がドラッグで完結する。伏線の登録→回収のループがプロットモードで回る。check.sh 全通過。

## UI-4: 執筆モードの磨き込み 【必須】

### Must

- [ ] **インスペクタ再編**: タブを [章メモ | この章 | 資料] に。「この章」タブ = 選択章に紐付くプロットカード一覧(クリックでプロットモードへ)+ 本文から検出した登場キャラ(クリックでキャラクターモードへ)。キャラ/プロットの編集本体は各モードにある前提の、読み取り中心のコンテキスト表示
- [ ] **設定ウィンドウ**(SwiftUI `Settings` scene): エディタのフォントサイズ(12〜24pt)/ 行間(1.2〜2.0)/ 本文の最大幅(制限なし / 700pt / 900pt 中央寄せ)。UserDefaults 保存。EditorKit の公開 API に `EditorConfiguration`(フォントサイズ・行間・最大幅の値型)を追加して EditorView に渡す — **NSTextView を公開 API に出さないこと(9.2)。設定変更の反映はテキスト所有権ルール(D-005)を破らないこと**(本文の再流し込みではなく、typingAttributes / defaultParagraphStyle / textContainer の更新で行う)
- [ ] **視覚調整**: アプリのアクセントカラーを落ち着いた藍系に統一(Assets の AccentColor)。エディタの `textContainerInset` を左右24ptに拡大。空状態(章なし/キャラなし/カードなし)の文言とアイコンを統一トーンに
- [ ] ステータスバーは執筆モード専用のまま維持(章/全体の文字数・原稿用紙換算)

### Nice to have

- [ ] 執筆モード時のフォーカス(サイドバー・インスペクタを同時に隠すショートカット)
- [ ] 選択章の変更を「この章」タブに即時反映するアニメーションの調整

### 完了条件

設定でフォント・行間・幅を変えて執筆でき、IME・自動インデント・undo にリグレッションがない(EditorKit の既存テスト+手動確認)。check.sh 全通過。
