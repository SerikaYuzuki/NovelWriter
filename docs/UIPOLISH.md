## UI-POL-1: インデント仕様の変更とバグ修正 【完了・エディタ核心】

### 症状(ユーザー報告)

1. 2連続で改行すると字下げが無効になる(現行 R2 の仕様どおりだが、望ましくない)
2. `「` を入力しても字下げ(全角スペース)が消えない

### 決定済みの新仕様(D-033)

- **R1'(常時字下げ)**: 改行は行の内容にかかわらず、常に `"\n\u{3000}"`(改行+全角スペース1つ)を挿入する。**旧 R2(空白のみ行での改行時に空白を掃除して字下げしない)は廃止**。字下げが不要な行は、ユーザーが自分でスペースを消す
- **R3(直接入力の字下げ解除)**: 現行仕様を維持 — 行の内容がちょうど `　` でキャレットが行末のとき `「` / `『` を入力すると、全角スペースを鉤括弧に置き換える
- **R5(IME確定後の字下げ解除)— 新設**: 症状2の根本原因は、日本語IMEで `「` を打つと変換(marked text)を経由するため、shouldChange ベースの R3 が一度も発火しないこと(IMEGuard が正しくパイプラインを止めている)。そこで **IME確定後の `textDidChange`(非変換中)で後処理**する:
  - 判定(純ロジックとして IndentRules に追加): キャレット直前の文字が `「` または `『`、その直前が**行頭の全角スペース1つ**(= 行が `　「` / `　『` で始まり、キャレットが鉤括弧の直後)なら、その全角スペースを削除するアクションを返す
  - 適用は MacTextAdapter の `textDidChange` 内、`hasMarkedText() == false` かつ既存の `isApplyingPluginReplacement` ガードの下で、**undo が効く正規経路**(shouldChangeText → textStorage → didChangeText)で行う。R4(IME変換中は不介入)とは矛盾しない(確定後にのみ動く)
  - ペースト等で同じパターンができた場合に適用されるのは許容(会話行を字下げしない作法に沿うため)。キャレット位置基準の局所判定のみとし、全文走査はしない

### 作業項目

- [x] `IndentRules` の改行判定から旧 R2 分岐を削除し、常時字下げに変更。旧 R2 のテストを新仕様のテストへ書き換え(空白のみ行で改行 → 字下げされる / 空白は掃除されない)
- [x] `IndentRules` に R5 判定の純関数を追加(UTF-16 range 安全、絵文字混在テスト付き)
- [x] `MacTextAdapter.textDidChange` に R5 適用を追加。**統合テスト**: `setMarkedText` → 確定挿入の流れで `　「` → `「` になること / **Undo で全角スペースが戻ること** / 直接入力の R3 が引き続き動くこと / R5 適用が `onTextChange` へ正しい全文を届けること(D-005 回帰)
- [x] DESIGN.md 4.5 の確定ルール(R1〜R4)を新仕様(R1' / R3 / R4 / R5)へ更新

**完了条件:** 「改行→改行→本文」で両行とも字下げされる。日本語IMEで `「` を確定した直後に行頭の字下げが消える(直接入力でも同様)。Undo・IME・検索ジャンプに回帰がない。

**実装結果 (2026-07-11):** 旧 R2 を廃止して R1'(常時字下げ)へ変更し、`IndentRules.postChangeAction` と `MacTextAdapter.textDidChange` で R5(IME確定後の字下げ解除)を追加した。統合テストで IME 確定・Undo・空白行改行を検証し、DESIGN 4.5 を R1'/R3/R4/R5 へ更新した。

## UI-POL-2: 傍点の記法変更(1文字ずつのルビ点方式) 【D-034】

### 決定済みの新仕様

現行(D-030-3)の `《《対象文字列》》` を**改訂**し、**選択範囲の1文字(grapheme)ごとに `｜字《・》` を生成**する方式にする(なろう互換。`《《》》` はカクヨム限定記法のため)。

例: 「例えばこの文章」を選択して傍点 →
`｜例《・》｜え《・》｜ば《・》｜こ《・》｜の《・》｜文《・》｜章《・》`

- 改行はそのまま残す(変換対象外)。半角/全角スペースもそのまま(傍点を振らない)
- それ以外の Character(絵文字含む)は1文字ずつ `｜C《・》` に変換。記号のescapeはしない(現行方針を踏襲)
- **入力シートは廃止**: 傍点は選択範囲の直接変換にする(選択が空のときはボタンを disabled)。ルビは従来どおり入力シートを使う
- 置換は既存の D-030 command 境界(選択スナップショット → `replaceSelection`)を使い、**1 Undo 単位**にする

### 作業項目

- [x] `EditorNotationRules.bouten` を新仕様へ書き換え + テスト(上記例文 / 絵文字1文字 / 改行・空白を含む選択 / 空選択で nil)
- [x] 傍点ボタンの動作をシート無しの直接変換へ変更(選択なしで disabled)。ルビ側は変更しない
- [ ] D-030 の記法規定は DECISIONS 上で「D-034 で改訂」と注記済み(本計画のPRで実装)

**完了条件:** 選択→傍点で上記例文どおりの出力になり、Undo 1回で元に戻る。IME変換中・選択消失時に本文が壊れない(既存の command 境界テストを踏襲)。

## UI-POL-4: ツールバーの再配置と棚卸し 【D-035。最後にやる(規模が大きい)】

### 目標配置(D-035 で決定済み)

「一覧アイテムの追加は Outline 上」「本文への操作は Editor 上」に統一する。

| 領域 | 項目 | 固定/カスタマイズ |
|---|---|---|
| Project Sidebar 上 | 標準の開閉 | 固定(不変) |
| Outline 上 | 作品名 + 章数(identity) | 固定(不変) |
| Outline 上 | **セクション連動の追加ボタン**: 執筆 = 章を追加 / プロット = 章を追加 / 登場人物 = 登場人物を追加 / 世界観 = ノートを追加 / 資料 = 資料を取り込む… | 固定 |
| Editor 上・**左端** | **話を追加**(`square.and.pencil`。執筆セクションのみ) | 固定(左揃え) |
| Editor 上・中央 | 話メモ / スナップショット / **プロットカードを追加**(プロットセクションのみ) | カスタマイズ可 |
| Editor 上・右端 | 話内検索 | 固定(不変) |

### 棚卸し(現行 → 変更)

| 現行アイコン | 処置 |
|---|---|
| `workbench.episode.add` 話を追加 | Editor 左端の固定へ移動 |
| `workbench.chapter.memo` 話メモ | 中央に残す |
| `workbench.snapshot.save` スナップショット | 中央に残す(File メニューにもあり) |
| `workbench.chapter.context` この章 | **削除**(Outline の文脈メニューとプロット/人物側のジャンプで代替できており、利用実態が薄い) |
| `workbench.character.add` 登場人物を追加 | Outline 上へ移動 |
| `workbench.plot.card.add` プロットカードを追加 | 中央に残す(**カード追加の導線はこのアイコンのみ**にする) |
| `workbench.attachment.add` 資料を取り込む… | Outline 上へ移動 |
| 執筆 Outline ペーン内の `+`(章を追加) | Outline 上のツールバーへ移動(ペーン内ボタンは撤去) |
| 世界観のノート追加ボタン | Outline 上のツールバーへ移動(ペーン内/現行位置の重複導線は撤去) |
